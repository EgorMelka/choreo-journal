# Журнал хореографа

Минималистичное адаптивное веб-приложение для учета посещаемости, доп-часа и расчета дохода.

## Вход по логину и паролю (Supabase)

Приложение теперь открывается только после авторизации `email + пароль`.

1. Создай проект в Supabase.
2. В Supabase открой `Project Settings -> API`.
3. Скопируй:
   - `Project URL`
   - `anon public key`
4. В корне проекта создай файл `.env.local`:

```env
VITE_SUPABASE_URL=https://YOUR_PROJECT.supabase.co
VITE_SUPABASE_ANON_KEY=YOUR_ANON_KEY
```

5. Перезапусти приложение:

```bash
npm run dev
```

## Логика денег

Считается только то, что реально было:

- день учитывается, только если включен `Пров`
- если включен `Отм`, доход = `0`
- будущие даты не считаются

Формула за занятие:

- `0-5` человек: `600 ₽`
- `6+`: `600 + 110 * (N - 5)`

`Доп` считается как второе занятие в этот же день по той же формуле.

## Что есть

- Разделы меню: `Журнал`, `Отчёты`, `Ученики`, `Группы`
- Стрелки даты назад/вперед + календарь
- Архив учеников (без потери истории)
- Карточка ученика: ФИО, дата рождения, авто-возраст
- Управление группами и расписанием
